## Background (why)

Root cause of "ghostty から herdr を実行しても 2 ペイン構成にならない": the bare-`herdr` routing lives in a zsh FUNCTION in `~/.zshrc`. Already-running shells never reload `.zshrc`, so shells started before the function landed bypass `herdr-agents` entirely and attach to a plain pane. The fix moves the entrypoint to a file on disk and wires Ghostty's official `initial-command` to it. Ghostty custom commands run via `/bin/bash --noprofile --norc` (no rc files, no mise shims on PATH), so the command must be wrapped in `/bin/zsh -lc`.

## Goal

1. **New file** `home/dot_local/bin/common/executable_herdr-session` (deploys to `~/.local/bin/common/herdr-session`), bash, shdoc annotations (`@file`, `@brief`, `@description`, `@example` — match the style of `executable_herdr-agents`). Behavior:
   - Run `herdr-agents "$PWD"`, appending BOTH stdout and stderr to `~/.config/herdr/herdr-agents.log` (failure visibility; the log dir already exists — herdr owns it. Also `tee` the output to the terminal is NOT required; keep it simple: redirect to log only).
   - If `herdr-agents` fails: print a one-line warning to stderr naming the log path, then continue (degraded attach, same as today's function).
   - `exec herdr` to attach (resolve via PATH; do NOT hardcode the mise install path).
   - No arguments supported; `--help`/`-h` prints short usage (match herdr-agents style). Any other argument → usage to stderr, exit 2.
2. **Edit** `home/dot_zshrc`: replace the body of the existing `herdr()` function (lines 21-32) so it is a thin delegate — keep exactly this semantics:

   ```zsh
   function herdr() {
       if [[ $# -eq 0 && -n "${GHOSTTY_RESOURCES_DIR:-}" ]]; then
           herdr-session
           return
       fi
       command herdr "$@"
   }
   ```

3. **Edit** `home/dot_config/ghostty/config`: add (with a brief comment, matching the file's comment style):
